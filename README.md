#  Music-Store-Data-Analysis-using-SQL
##  Project Overview

The Music Store Data Analysis project focuses on analyzing a digital music store database using SQL to extract meaningful business insights.

The database contains structured information about customers, invoices, tracks, albums, artists, genres, and sales transactions. The objective of this project is to analyze customer behavior, sales performance, and revenue trends using advanced SQL queries.

This project demonstrates practical application of SQL in solving real-world business problems.

---

##  Objectives

- Analyze sales performance across different countries and cities  
- Identify top-selling genres, artists, and tracks  
- Understand customer purchasing behavior  
- Calculate revenue trends and customer spending patterns  
- Generate actionable insights to support business decisions  

---

##  Dataset Description

The dataset consists of multiple relational tables:

- **Customers** – Customer details and geographical information  
- **Invoices & InvoiceLine** – Transaction and revenue data  
- **Tracks** – Song details including pricing and duration  
- **Albums & Artists** – Music catalog information  
- **Genres** – Music categories  
- **Employees** – Store staff information  

The database follows a normalized relational schema with primary and foreign key relationships.

---

##  Tools & Technologies Used

- MySQL  
- Joins (INNER JOIN)  
- Subqueries  
- Aggregate Functions (SUM, COUNT, AVG)  
- GROUP BY & ORDER BY  
- CTEs (Common Table Expressions)  
- Window Functions (RANK, DENSE_RANK)  

---

##  Key Insights

- A small group of customers contributes significantly to overall revenue.  
- Certain genres dominate sales across multiple regions.  
- The USA generates the highest number of sales invoices.  
- Popular artists drive repeated customer purchases.  
- Revenue concentration varies significantly by country and city.
